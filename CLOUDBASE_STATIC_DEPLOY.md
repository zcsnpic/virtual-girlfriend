# 腾讯云 CloudBase 静态网站托管部署指南

## 问题分析

你遇到的错误：`✖ Path does not exist: /root/cloudbase-workspace/dist` 是因为你可能选择了 **自定义部署** 模式，并指定了构建命令，但我们的项目是静态网站，不需要构建过程，直接上传文件即可。

## 正确的部署方法

### 方法一：直接上传文件（推荐）

1. **进入静态网站管理**
   - 登录腾讯云控制台，进入云开发 CloudBase
   - 选择你的环境，进入 **静态网站** 管理页面

2. **直接上传文件**
   - 点击 **上传文件** 按钮
   - 选择以下文件和文件夹：
     - `index.html`
     - `css/` 文件夹
     - `js/` 文件夹
   - 确保文件结构正确

3. **配置网站设置**
   - 设置索引文档为 `index.html`
   - 设置错误文档为 `index.html`
   - 保存配置

4. **访问应用**
   - 复制默认域名，在浏览器中访问

### 方法二：使用压缩包上传

1. **创建压缩包**
   - 选中 `index.html`、`css/` 和 `js/`
   - 右键选择 **发送到** > **压缩(zipped)文件夹**
   - 命名为 `website.zip`

2. **上传压缩包**
   - 在静态网站管理页面，点击 **上传文件**
   - 选择创建的 `website.zip` 文件
   - CloudBase 会自动解压

3. **配置并访问**
   - 按方法一的步骤配置网站设置
   - 访问默认域名

## 避免使用自定义部署

**不要选择自定义部署**，因为：
- 我们的项目是纯静态网站，不需要构建
- 自定义部署会执行构建命令，寻找dist目录
- 我们的项目结构中没有dist目录

## 部署验证

部署完成后，测试以下功能：
1. 对话功能是否正常
2. 记忆功能是否工作
3. 记忆管理界面是否可访问
4. 响应式设计是否正常

## 常见问题

### 1. 上传后访问404
- **解决**：检查文件结构，确保 `index.html` 位于根目录

### 2. 功能异常
- **解决**：检查浏览器控制台是否有JavaScript错误

### 3. 页面加载缓慢
- **解决**：开启CDN加速选项

## 部署完成

按照以上步骤部署后，你应该能够正常访问虚拟女友应用。如果仍有问题，请检查网络连接或联系腾讯云技术支持。
