# 性能优化功能使用指南

**版本**: v4.1.2
**日期**: 2026-03-24
**功能**: 语音和情景文字播放性能优化

---

## 功能说明

为了解决 AI 消息拆解后播放节奏缓慢的问题，新增了**双逻辑并存模式**。

### 模式一：原有串行模式（默认，安全）

**特点**：
- 场景显示 800ms → 场景隐藏 → 语音播放
- 完全保留原有行为
- 100% 兼容现有功能

**适用场景**：
- 第一次测试，确保原有功能正常
- 对新优化不确定时使用
- 需要与旧版本对比时使用

---

### 模式二：并行优化模式（推荐）

**特点**：
- 场景显示 → 立即播放语音 → 400ms 后淡出场景
- 场景和语音并行处理，减少等待时间
- 节省约 30% 的播放时间

**适用场景**：
- 已测试过原有模式，确认功能正常
- 希望提升播放速度
- 对体验有更高要求

---

## 如何切换模式

### 方式一：修改代码（开发者）

1. 打开文件：`js/app.js`

2. 找到第 11 行：
   ```javascript
   // 性能优化开关：false=原有逻辑（安全），true=新优化逻辑
   useParallelSceneAndSpeech: false,
   ```

3. 修改值：
   - 原有串行模式：`useParallelSceneAndSpeech: false`
   - 并行优化模式：`useParallelSceneAndSpeech: true`

4. 保存文件，刷新浏览器即可生效

---

### 方式二：浏览器控制台（快速测试）

1. 打开浏览器控制台（F12）

2. 输入以下命令：
   ```javascript
   // 切换到并行优化模式
   App.useParallelSceneAndSpeech = true;
   console.log('已切换到并行优化模式');
   
   // 切换回原有串行模式
   App.useParallelSceneAndSpeech = false;
   console.log('已切换回原有串行模式');
   ```

3. 发送一条消息测试效果

**优势**：无需修改文件，立即生效，随时切换

## 技术细节

### 原有串行模式流程

```
消息1: [场景] 显示 800ms → 隐藏 → 播放语音 (2000ms) → 等待 200ms
消息2: [场景] 显示 800ms → 隐藏 → 播放语音 (3000ms) → 等待 200ms
消息3: [场景] 显示 800ms → 隐藏 → 播放语音 (1500ms)
总耗时: 9.3 秒
```

### 并行优化模式流程

```
消息1: [场景] 显示 → 播放语音 (2000ms) → 400ms 后隐藏场景 → 等待 200ms
消息2: [场景] 显示 → 播放语音 (3000ms) → 400ms 后隐藏场景 → 等待 200ms
消息3: [场景] 显示 → 播放语音 (1500ms) → 400ms 后隐藏场景
总耗时: 6.5 秒
```

### 改进原理

1. **场景不再等待** - 显示场景后立即开始播放语音
2. **场景淡出提前** - 400ms 后淡出，而不是 800ms
3. **并行处理** - 场景淡出与语音播放重叠

## 常见问题

### Q1: 优化模式会影响语音质量吗？

**A**: 不会。优化只调整时间控制，语音播放逻辑完全不变。

### Q2: 场景会显示得太短吗？

**A**: 不会。400ms 是淡出时间，场景在语音开始后作为背景存在，用户注意力转移到语音，场景淡出更自然。

### Q3: 两个模式可以同时存在吗？

**A**: 是的，代码中同时保留了两套逻辑，通过开关控制，随时切换。

### Q4: 切换模式需要重启应用吗？

**A**: 不需要。通过控制台切换立即生效，修改文件后刷新浏览器即可。

### Q5: 如果优化模式有 Bug 怎么办？

**A**: 立即通过控制台或修改文件切回 `false`，原有模式不受影响。
